# Phase 2: 量化选股脚本开发 (Quantitative Screening) 执行计划与总结

## 目标

实现一个独立的 Python 脚本 (`scripts/quant_picker.py`)，该脚本基于指定的 T 日（月末或月初），从本地 PostgreSQL 的 `kline_qfq` (日K线表) 中提取过去一个月的历史数据，通过设定的技术特征进行打分排序，最终输出符合条件的 TOP-20 股票池，供下一阶段 OpenClaw 监控使用。

## 选股特征规则（标准化定义）

### 1. 时间与数据定义

- **选股执行日**：指定 `T` 日。
- **历史滑动窗口搜索**：为了避免单个月份固定窗口带来的局限性，在寻找「起涨-回调-回抽」形态时，**引入多窗口滑动机制**。
  - 从指定的 `T` 日开始，每次往前取一个月的窗口（22个交易日），比如：
    - 窗口 1: `[T-22, T]`
    - 窗口 2: `[T-44, T-22]`
    - 窗口 3: `[T-66, T-44]` 
    - 依此类推，可设置滑动找寻过去 `N` 个月（例如 N=3，即过去一季度）的数据。
  - **目的**：只要在这个滑动的漫长观察期内，**成功走出一个符合我们 A->B->C 结构的波段，且该结构距离当前 T 日不过于久远（避免行情过期）**，我们就将其捕获。
- **基础过滤**: 排除 ST、*ST、退市股票，以及上市时间过短的新股。

### 2. 关键 K 线结构的「标准化定义」

#### 节点 A：「放量大涨」（攻击浪起点）

- **涨幅条件**:
  - 创业板/科创板 (代码 300/688 开头)：单日最高涨幅 ≥ 12%
  - 主板（非创业/科创）：单日最高涨幅 ≥ 7%
  - *优先条件*: 出现连板或连续 2 天满足上述涨幅（强势股标志）。
- **放量条件**:
  - 当日成交量 $V_A \ge MA(V,5) \times 1.5$（即比前 5 天均量放大 50% 以上）
- **记录**: 记下这根 K 线的日期 `Date_A` 和收盘价 `Close_A`。

#### 节点 B：「深度回调」（洗盘最低点）

- **时间范围**: 必须发生在 `Date_A` 之后。
- **跌幅条件** (满足任意一条即视为有效洗盘，满足越多越强势):
  1. **连续下跌**: 出现连续 1-5 根阴线（收盘价 < 前一日收盘价）。
  2. **大阴线/跌停**: 单日跌幅 ≥ 5%，或直接触及跌停板。
  3. **破位均线**: 收盘价 $C_B < MA25_B$（有效跌破 25 日均线）。
- **记录**: 记下洗盘的最低点日期 `Date_B` 和最低价 `Low_B`。

#### 节点 C：「快速回抽」（确认信号，核心中的核心）

- **时间限制**: 必须在 `Date_B` 之后的 3 个交易日内发生（即 $Date_C - Date_B \le 3$）。
- **强度条件** (满足任意一条即有效，满足越多优先级越高):
  1. **涨停确认**: 直接涨停（最强信号）。
  2. **跳空缺口**: 当日开盘价 $Open_C > Close_{C-1} \times 1.02$（高开 2% 以上，且缺口不回补）。
  3. **大阳反攻**: 单日涨幅 ≥ 5%，且收盘价收复前 3 日跌幅的 50% 以上。
- **幅度检验（新增强化规则）**:
  - 核心逻辑：回抽的幅度是否能挑战乃至突破前期高点（即节点 A `Close_A` 或 `High_A`）。
  - **加分/减分项**：
    - 如果回抽点 `Date_C` 及其后几日的最高价 $High \ge Close_A$（甚至突破了前高），这代表洗盘结束，开启二波主升浪，**大幅加分**。
    - 如果回抽仅仅是弱反弹（例如连 $Close_A$ 的一半都没摸到），则视为弱势反抽，不仅不加分，还要**大幅减分**甚至淘汰。
- **记录**: 记下回抽日的日期 `Date_C`。

### 3. 「涨幅天花板」过滤器（防止接盘）

- **计算基准**: 从 `Date_A` 的前一天（起涨前）开始计算。
- **限制条件**:
  - 创业板/科创板：区间累计涨幅 ≤ 60%
  - 主板：区间累计涨幅 ≤ 35%
- **目的**: 过滤掉已经涨上天的“鱼尾”行情，只吃“主升浪中途休息”的那段。

## 目前完成情况

Phase 2 的核心代码 `scripts/quant_picker.py` 已经开发完毕并经过初步测试。
主要实现了以下高级特性：
1. **严格的节点识别**：精确实现了节点 A（放量大涨）、节点 B（深度回调）和节点 C（快速回抽）的量化逻辑。
2. **回抽幅度检验**：在节点 C 中加入了对回抽高点是否能突破/触及节点 A 收盘价的严格检验，弱反抽会被直接淘汰或大幅扣分。
3. **滑动窗口机制**：不再局限于单一的 T-22 窗口，而是默认往前滑动 3 个窗口（约 66 个交易日），只要在这期间走出了标准的 A->B->C 形态即可被捕获，且越近期的形态得分权重越高。
4. **5日风控（涨幅天花板）**：严格计算了起涨点前 5 日的累计涨幅，防止选入已经处于高位“鱼尾”行情的股票。

## 接下来的执行与测试计划 (人工校验)

为了确保选出的 TOP-K 股票符合您的预期，请按照以下步骤进行人工校验，我们暂时**不急于进入 Phase 3**：

1. **运行脚本生成结果**：
   您可以打开终端，运行以下命令来执行选股脚本。如果不指定 `--t_date`，脚本默认会使用**当前日期**（今天）作为基准往前推算；您也可以像下面这样显式指定一个过去的日子作为 T 日：
   ```bash
   python scripts/quant_picker.py --t_date 2024-03-01
   ```

2. **查看输出结果**：
   脚本会在控制台打印出得分最高的 TOP-20 股票，并包含详细的得分、各个节点的日期以及 5日风控数据。同时，结果会保存在 `results/watchlist_YYYY-MM-DD.json` 文件中。

3. **人工复盘校验**：
   - 打开您的炒股软件（如通达信、同花顺等）。
   - 按照脚本输出的股票代码和对应的 `Date_A` (大涨日)、`Date_B` (见底日)、`Date_C` (回抽日)，在 K 线图上进行人工核对。
   - 重点观察：
     - A 点是否真的是放量大涨的启动点？
     - B 点的洗盘是否充分？
     - C 点的回抽是否强劲有力？
     - 整体形态是否符合您心中的“主升浪中途休息”的预期？

4. **反馈与参数微调**：
   - 如果您发现某些选出来的股票形态很差（比如回抽太弱、或者前期已经涨了太多），请告诉我具体的股票代码，我们可以一起分析是哪个规则不够严格，并调整 `scripts/quant_picker.py` 中的参数（例如提高回抽分数的门槛、收紧风控比例等）。
   - 如果您发现某些您认为非常符合形态的股票**没有**被选出来，也请告诉我代码和您认为的 T 日，我们可以通过打印该股票的单步评分日志，找出它是被哪条规则“误杀”的。

等您人工校验满意，确认 TOP-K 排名的准确性后，我们再正式开启 Phase 3（OpenClaw 监控与 LLM 分析）的开发。
