## 1. 目标与边界

### 1.1 目标

- 实现独立 Python 能力（建议脚本路径：`scripts/phase4_weekly_screener.py`，名称可确认后定稿）：
  - 读取前复权日 K（`kline_qfq`）与交易日历（`trading_calendar`）。
  - **内存中**生成周 K（v1 **不新建周 K 线物理表**）。
  - 按可配置规则识别：**历史大顶 → 长周期下跌（回撤与时长）→ 筑底区间 → 启动触发**。
  - 输出候选列表（JSON/CSV，字段见下文），便于全市场多次试跑与 2025/2026 数据回测调参。
- 与 Phase 3 衔接（后续迭代）：将候选池或「仅启动触发」子集写入 OpenClaw 监控列表，结合实时行情做提醒（本阶段可先只完成筛选与输出）。

### 1.2 边界与非目标（v1）

- 不承诺「绝对真底」，以**多条件概率提升**为原则；参数外置，便于回测后调整。
- v1 **不建** `kline_week_qfq` 等周 K 表；若后续全市场性能不足，再单列「物化周 K 表」迭代（见第 8 节）。
- 周线上的 MA120/MA250、VOL_MA、放量指标均在脚本内由周 K 序列计算；**不使用** `kline_qfq` 中日线口径的 `ma5/ma10/ma20/ma60` 充当周线指标。

## 2. 数据依赖

| 来源 | 用途 |
|------|------|
| `kline_qfq` | 日级 OHLCV（及可选 amount），前复权一致性依赖现有更新脚本 |
| `trading_calendar` | 仅交易日参与聚合；周界按自然周内交易日划分（你已维护 2019–2026） |
| `stock_list` | 代码与名称；过滤 ST、退市等（规则与 Phase 2 对齐或可配置） |

**数据质量前提**：全市场可比区间建议有统一起点（例如不早于 2019-01-01）；不足最短历史长度的标的跳过或标记 `insufficient_history`。

## 3. 周 K 生成规则（v1 标准）

- 使用 `trading_calendar` 中 `is_trading_day = true` 的日期与 `kline_qfq.date` 交集。
- **周划分**：按**自然周**（ISO 周或「周一–周日」内出现的交易日归同一周；具体实现选一种并在代码注释与本文档保持一致）。
- **周 K 字段**：
  - `week_end_date`：该周**最后一个交易日**（作为该周 K 的日期索引）。
  - `open`：周内第一个交易日 `open`
  - `high`：周内 `high` 的最大值
  - `low`：周内 `low` 的最小值
  - `close`：周内最后一个交易日 `close`
  - `volume`：周内日 `volume` 之和
  - `amount`：若日 K 有可靠 `amount`，可同期求和（可选）

## 4. 形态与参数（v1 默认值，可配置）

以下数值建议写入配置文件或脚本顶部常量，便于回测时批量扫描调整。

### 4.1 历史峰值与回撤（下跌深度）

- 在足够长回溯窗口内（默认：**数据可用全量**，且不少于 N 年，N 可配置如 5）找周线收盘价**最高点**及所在周 `T_peak`。
- **回撤**：从 `T_peak` 之后某「底部参考价」（如筑底区间最低价，或启动前若干周最低价）相对峰值跌幅，落在 **50%～80%**（闭区间或半开，实现时统一约定）。

### 4.2 下跌时长

- 从 `T_peak` 到进入「筑底」或到「启动周」之前，满足**长熊结构**的持续时间：**1.5～4 年**。
- 实现上可换算为**交易周数区间**（约 78～208 周，按每年约 52 周估算；精确以「满足条件的连续周计数」为准）。

### 4.3 长熊结构（v1 可操作定义）

至少满足其一或组合（具体组合方式开发时实现为可配置「AND/OR」）：

- 峰值后较长阶段：周线收盘在 **MA60（周）** 下方时间占比超过阈值（待定默认，如 >50%）。
- 或：峰值后存在明显的 **lower high** 序列（简化实现可用「滚动 N 周高点逐步下移」判定）。

### 4.4 筑底（v1）

作为第一版，采用此前讨论的可量化 proxy，**后续完全由回测结果调整**：

- **波动收敛**：筑底窗口内，周 `(high-low)/close`（或振幅）相对下跌段 **分位数下降**（阈值待定）。
- **均线粘合**：周 `MA5/10/20/60` 相对收盘价的离散度（如变异系数或最大最小比）**低于**阈值（待定）。
- **时长**：筑底平台持续 **最少 K 周**（待定默认，如 20～40 周量级，可配置）。

### 4.5 启动（v1）

同样 v1 固化 + 回测迭代：

- **收盘站上长均线**：启动周（或连续 2 周内）周收盘 **> MA120** 或 **> MA250**（周线上计算；二者可配置为「满足其一」或「优先 MA250」）。
- **短均线拐头**：辅助条件，如周 `MA5 > MA20` 或金叉（可选开关）。

### 4.6 成交量均线

- 对**周成交量**计算 **VOL_MA5 / VOL_MA10 / VOL_MA20**（均为「周」维度下的均线）。

### 4.7 放量（组合方案，v1）

同时满足结构前提下：

- **相对自身**：\( VR = \text{当周成交量} / \text{VOL\_MA20} \)，阈值默认 **≥ 1.5～2.0**（可配置）。
- **相对筑底期**：筑底区间内周成交量**中位数** `VOL_med_base`，启动周 **≥ k × VOL_med_base**，k 默认 **2～3**（可配置）。

实现时要求：**VR 条件与 k 条件同时满足**（与你确认的「组合方案」一致）。

## 5. 输出约定（v1）

建议每张候选至少包含：

- `instrument`, `stock_name`
- `T_peak`, `peak_close`, `drawdown_ratio`
- `base_start`, `base_end`（筑底区间，若算法可标定）
- `trigger_week`（满足启动条件的 `week_end_date`）
- `vr`, `vol_med_base`, `close_vs_ma120`, `close_vs_ma250`
- `score`（可选：多条件加权，v1 可先 0/1 再演进）
- `notes` / `fail_reason`（未入选时的调试字段，可选）

输出路径：例如 `output/phase4_candidates_{as_of_date}.json`（与 Phase 2 输出习惯对齐）。

## 6. 主流程（实现顺序建议）

1. **周 K 聚合模块**：输入单股日 K + 全局交易日列表 → 输出周 K DataFrame。
2. **周线指标模块**：MA5/10/20/60/120/250，VOL_MA 系列。
3. **单股状态机 / 规则引擎**：峰值定位 → 回撤与时长 → 筑底判定 → 启动与放量判定；返回结构化结果或失败原因。
4. **全市场循环**：批量读取（可按 instrument 分批，控制内存）、过滤 ST、记录日志。
5. **CLI**：`--as-of`（截至某交易日或某周结束日）、`--instruments` 子集调试、`--config` 可选。
6. **单股黄金校验**：使用 `002865.SZ`（数据补全后）人工对照软件周 K，校准周界与阈值。

## 9. 风险与待定项

- **前复权**：历史分段补数需同一复权体系；除权后若与日更脚本逻辑冲突，依赖现有 `update_kline_qfq` 检测与全量重拉。
- **周界与软件差异**：与券商软件周 K 可能差一根边界，接受小幅偏差，全市场口径一致即可。
- **待定默认阈值**：筑底最小时长、波动/粘合精确公式、长熊「MA60 下方占比」等，在编码阶段给合理初值，**以你回测反馈为主**。
