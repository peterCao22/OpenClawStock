# Phase 2 优化方案：基于动态窗口的形态识别开发计划

## 1. 目标 (Objective)

本阶段的目标是优化量化选股逻辑，引入**动态时间窗口**机制，将过去约 4 个月的 K 线走势划分为 4 个基于价格行为（Price Action）的自然区间。通过分析这 4 个区间的涨跌特征和相对位置，识别出 **N型、W底、H型、V底** 四种主力资金运作的强势形态。

此逻辑将作为**第一层宏观筛选 (Macro Filter)**，筛选出的股票再结合原有的 A->B->C 细节逻辑进行最终确认。

---

## 2. 核心算法逻辑

### 2.1 动态窗口分割 (Dynamic Window Segmentation)

不再使用固定的 T-22, T-44 切分，而是根据极值点动态回溯。

*   **输入**: T 日 (End Date), 历史 K 线数据 (约 150 天)
*   **输出**: 4 个区间对象列表 `[Window1, Window2, Window3, Window4]` (W1 为最近，W4 为最远)
*   **逻辑**:
    1.  **Window 1**:
        *   `End_Date` = T
        *   `Start_Date` = T - 22 交易日
        *   在 `[Start, End]` 范围内找 `High_Price` (日期 `Date_H`) 和 `Low_Price` (日期 `Date_L`)。
        *   **关键**: 下一个窗口的 `End_Date` = `min(Date_H, Date_L)` (即取更早发生的那个极值日期)。
    2.  **Window 2**:
        *   `End_Date` = Window 1 的 `Next_End`
        *   `Start_Date` = `End_Date` - 22 交易日
        *   找 H2, L2, Date_H2, Date_L2。
        *   `Next_End` = `min(Date_H2, Date_L2)`。
    3.  **Window 3 & 4**: 重复上述逻辑。

### 2.2 区间特征提取 (Feature Extraction)

对每个窗口 (Window)，计算以下关键特征：

1.  **极值**: `High`, `Low`, `Date_H`, `Date_L`
2.  **区间方向**:
    *   若 `Date_L < Date_H`: **上涨波段** (Low -> High)
    *   若 `Date_H < Date_L`: **下跌波段** (High -> Low)
3.  **波段涨跌幅**:
    *   上涨波段: `(High - Low) / Low`
    *   下跌波段: `(Low - High) / High` (负值)
4.  **K线结构**:
    *   阳线数量 / 阴线数量
    *   最大连涨天数 / 最大连跌天数
    *   涨停/大涨次数 (创业板>12%, 主板>6%)
    *   跌停/大跌次数 (创业板<-12%, 主板<-6%)

## 3. 形态识别逻辑 (Pattern Recognition)

基于提取的 4 个窗口特征 (W4 -> W3 -> W2 -> W1)，定义四种形态。

### 3.1 N 型 (最强攻击形态)
*   **逻辑描述**: 拉升(W4/W3) -> 洗盘(W2) -> 再次拉升(W1)
*   **判定条件**:
    1.  **前期拉升**: W4 或 W3 必须是显著的**上涨波段** (涨幅: 创业板>25%, 主板>15%)。
    2.  **中期洗盘**: W2 是**下跌波段** 或 **震荡** (跌幅可深可浅，但不能跌破 W4/W3 的启动点太多)。
    3.  **近期启动**: W1 是**上涨波段**，且 W1 的收盘价/最高价 接近或超过 W2 的起点 (即收复失地)。
    4.  **高位特征**: W1 的 `High` >= W3/W4 的 `High` (或接近，如 > 90%)。

### 3.2 W 底 (双底反转)
*   **逻辑描述**: 下跌 -> 反弹 -> 二次探底(不创新低) -> 拉升
*   **判定条件**:
    1.  **左底**: W3 或 W2 形成第一个低点 `L_Left`。
    2.  **右底**: W1 或 W2 形成第二个低点 `L_Right`。
    3.  **不创新低**: `L_Right` >= `L_Left` * 0.95 (允许微小刺破，但最好抬高)。
    4.  **颈线突破**: W1 的走势呈现**上涨波段**，且重心上移。

### 3.3 H 型 (高位横盘/强者恒强)
*   **逻辑描述**: 大涨后在高位长时间横盘，不跌，最后再次抬头。
*   **判定条件**:
    1.  **旗杆**: W4 或 W3 有巨大涨幅 (如: 创业板>35%, 主板>20%)。
    2.  **平台**: W2 和 W1 的价格区间 (`High` - `Low`) 相对较小，形成箱体。
    3.  **支撑**: W2 和 W1 的 `Low` 远高于 W4 的启动点 (通常在 W4 高点附近震荡)。
    4.  **突破**: W1 的收盘价接近箱体顶部。

### 3.4 V 底 (超跌反转)
*   **逻辑描述**: 暴跌后直接V型反转。
*   **判定条件**:
    1.  **暴跌**: W2 (或 W2+W3) 是剧烈的**下跌波段** (跌幅深，阴线多)。
    2.  **极值**: 最低点出现在 W2 的末尾或 W1 的开头。
    3.  **反转**: W1 是强劲的**上涨波段** (涨幅大，阳线多，有大阳/涨停)。
    4.  **斜率**: W1 的上涨斜率陡峭。

---

## 4. 风险控制 (Risk Control)

在识别形态的同时，必须剔除高风险目标：

1.  **短期过热**: W1 虽然是上涨，但如果累计涨幅过大 (如: 创业板>65%, 主板>40%) 且出现连续加速 (如 6 连板)，视为鱼尾行情，剔除 (除非是 H 型突破)。
2.  **下降趋势**: W4 -> W1 重心持续下移，且 W1 只是微弱反弹 (跌幅的一半都没收复)，剔除。

1.  验证能否正确切分出文档中描述的 4 个区间。

3.  验证是否触发“短期过热”风控提示。
