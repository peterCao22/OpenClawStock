# Phase 2 量化选股结果字段说明与形态对照表

本文档详细说明了 `results/*.json` 输出文件中的字段含义，以及 N/W/H/V 四种形态与 W1-W4 窗口组合的对应关系。

## 1. JSON 结果字段说明

| 字段名 | 类型 | 含义 | 示例 |
| :--- | :--- | :--- | :--- |
| `instrument` | String | 股票代码 (带后缀) | "002085.SZ" |
| `name` | String | 股票名称 | "万丰奥威" |
| `total_score` | Float | 综合评分 (越高越好) | 102.21 |
| `surge_date` | String | 最近一次放量大涨日 (节点A) | "2024-02-08" |
| `bottom_date` | String | 最近一次回调见底日 (节点B) | "2024-02-28" |
| `rebound_date` | String | 最近一次回抽确认日 (节点C) | "2024-02-29" |
| `cum_return_5d` | Float | 起涨前5日累计涨幅 (用于风控) | 8.38 (%) |
| `pattern` | String | 识别出的主力形态 | "W底: 双底结构" |
| `windows_info` | String | W1 窗口简要信息 | "W1:UP(58.1%)" |
| `windows_detail` | List | W1-W4 窗口详细数据 | (见下表) |

### `windows_detail` 列表字段

| 字段名 | 含义 | 说明 |
| :--- | :--- | :--- |
| `window` | 窗口标签 | W1 (最近) -> W2 -> W3 -> W4 (最远) |
| `role` | 形态角色 | 该窗口在当前形态中扮演的角色 (如 "Surge", "Crash") |
| `start_date` | 起始日期 | 该波段的开始时间 |
| `end_date` | 结束日期 | 该波段的结束时间 (通常是极值点) |
| `direction` | 波段方向 | "UP" (上涨) 或 "DOWN" (下跌) |
| `amplitude` | 波段幅度 | 该波段内的累计涨跌幅 (%) |

## 2. 窗口组合角色与形态对应表

OpenClaw 通过分析过去 4 个动态窗口 (W4 -> W3 -> W2 -> W1) 的组合特征来识别形态。

### 2.1 N 型 (强势整理后再起)

**核心逻辑**: 拉升 -> 洗盘 -> 再次拉升

| 窗口 | 角色 (Role) | 方向 | 幅度要求 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **W4 / W3** | **Surge (Start)** | **UP** | > 15% (主) / > 25% (创) | 第一波启动浪，确立趋势 |
| **W2** | **Wash (Pullback)** | DOWN/UP | - | 洗盘或震荡整理，不破前低 |
| **W1** | **Surge (Second Wave)** | **UP** | - | 第二波主升浪，收复失地，创新高 |

*(注：W4 或 W3 只要有一个满足 Surge 即可)*

### 2.2 W 底 (双底结构)

**核心逻辑**: 下跌 -> 反弹 -> 二次探底 -> 突破。代码中通过比较两个低点价格是否接近 (<10% 差异) 来判定。

#### 情况 A: 宽底结构 (W3 + W1)

| 窗口 | 角色 (Role) | 方向 | 说明 |
| :--- | :--- | :--- | :--- |
| W4 | Context | - | 背景趋势 (通常为下跌或震荡) |
| **W3** | **Left Bottom** | DOWN | **左底**：形成第一个低点 |
| **W2** | **Mid-Rebound** | UP | **颈线**：两底之间的反弹与回落过程 |
| **W1** | **Right Bottom/Breakout** | **UP** | **右底**：再次探底不破 W3 低点，并强力拉升 (**幅度 > 5% 且收复 50% 颈线跌幅**) |

#### 情况 B: 窄底结构 (W2 + W1)

| 窗口 | 角色 (Role) | 方向 | 说明 |
| :--- | :--- | :--- | :--- |
| W4 / W3 | Context | - | 背景趋势 |
| **W2** | **Left Bottom** | - | **左底**：近期形成的第一个低点 |
| **W1** | **Right Bottom/Breakout** | **UP** | **右底**：紧接着 W2 再次探底并拉升 (**幅度 > 5% 且收复 50% 颈线跌幅**) |

---

### 2.3 H 型 (高位横盘强者恒强)

**核心逻辑**: 巨幅拉升 -> 高位箱体震荡 -> 突破

| 窗口 | 角色 (Role) | 方向 | 幅度要求 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **W4 / W3** | **Flag Pole** | **UP** | > 20% (主) / > 35% (创) | 旗杆，前期巨大涨幅 |
| **W2** | **Platform** | - | - | 高位平台震荡，重心不因大盘下跌而下移 |
| **W1** | **Breakout/Platform** | **UP** | > -20% | 继续维持高位或突破，**严禁大跌破位** |

### 2.4 V 底 (超跌反转)

**核心逻辑**: 暴跌 -> 暴力反转

| 窗口 | 角色 (Role) | 方向 | 幅度要求 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **W2** | **Crash** | **DOWN** | < -15% | 恐慌性杀跌，制造黄金坑 |
| **W1** | **Reversal** | **UP** | > 跌幅的一半 | 报复性反弹，V型右侧确立 |
| W3 / W4 | Context | - | - | 背景趋势 (通常也是下跌或震荡) |

## 3. Role 角色字典 (中英对照)

| Role (英文) | 中文解释 | 对应形态 | 含义 |
| :--- | :--- | :--- | :--- |
| **Surge (Start)** | 启动浪 | N型 | 第一波确立趋势的上涨 |
| **Wash (Pullback)** | 洗盘/回调 | N型 | 上涨后的获利盘清洗，通常缩量 |
| **Surge (Second Wave)** | 主升浪/二波 | N型 | 洗盘结束后的再次拉升，力度往往更强 |
| **Flag Pole** | 旗杆 | H型 | 前期巨大的涨幅，如同旗杆一样竖立 |
| **Platform** | 平台 | H型 | 在高位构筑的箱体，重心不乱 |
| **Breakout** | 突破 | H型/W底 | 突破平台或颈线，开启新空间 |
| **Crash** | 暴跌 | V底 | 短期内的恐慌性杀跌 |
| **Reversal** | 反转 | V底 | 暴跌后的报复性反弹 |
| **Left Bottom** | 左底 | W底 | 双底形态的第一个低点 |
| **Mid-Rebound** | 中间反弹 | W底 | 连接两个低点中间的反弹波段 |
| **Right Bottom** | 右底 | W底 | 双底形态的第二个低点，确认支撑 |
| **Context** | 背景/上下文 | 所有 | 不构成核心形态的前序走势 |

## 4. 示例解读 (以 002085 万丰奥威 为例)

**形态**: W底: 双底结构

*   **W4 (Context)**: DOWN (-11.64%) -> 下跌背景
*   **W3 (Left Bottom)**: DOWN (-11.64%) -> **左底形成**
*   **W2 (Mid-Rebound)**: UP (30.36%) -> 中间反弹
*   **W1 (Right Bottom/Breakout)**: UP (58.12%) -> **右底确认并强力突破**

通过 W3 和 W1 的配合，清晰地画出了一个底部抬高的 W 形态。
