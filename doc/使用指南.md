# OpenClawStock 使用指南

本文说明日常运维：**数据库更新 → 选股 → 验证分析**。所有命令均在**项目根目录**执行，并使用 Conda 环境 `openclaw_env`。

```bash
conda activate openclaw_env
```

环境变量见根目录 `.env`（参考 `.env.example`）：PostgreSQL 连接、`MOMA_API_KEY` 等。

---

## 一、数据库表一览

| 表名 | 用途 | 主要更新脚本 |
|------|------|----------------|
| `trading_calendar` | 交易日历 | `scripts/sync_exchange_calendar.py` |
| `stock_list` | 股票列表 | `scripts/sync_moma_data.py --stock-list` |
| `kline_qfq` | 个股前复权日 K（选股/回测核心） | `scripts/sync_sohu_data.py`（推荐）或 `scripts/update_kline_qfq.py`（Moma） |
| `kline_qfq_monthly` | 月线派生表（含 `ma60_m`） | `scripts/build_kline_qfq_monthly.py`（随日 K 增量刷新） |
| `stock_dividend_event` | 分红送配事件（搜狐缓存） | `scripts/sync_sohu_data.py` / 一次性灌库 |
| `stock_dividend_meta` | 每股分红信息最近拉取时间 | 同上 |
| `index_bar1d` | 指数日线（上证/沪深300/深证成指） | `scripts/sync_sohu_data.py --index` |
| `limit_up_pool` 等 | 涨跌停、分类等辅助表 | `scripts/sync_moma_data.py` |

**口径说明**

- `kline_qfq` 成交量单位为**股**；搜狐接口为「手」，入库时 ×100。
- `kline_qfq` 价格为**前复权**；搜狐 hisHq 为不复权，经 `stock_dividend_event` 自算复权因子后写入。
- `index_bar1d` 成交量单位为**手**，成交额为**万元**。

## 二、日常数据更新（推荐顺序）

每个交易日收盘后（或次日开盘前）执行。

### 2.1 指数 + 全市场日 K（搜狐，推荐）

```bash
# 指数（上证 000001.SH / 沪深300 / 深证成指）
python scripts/sync_sohu_data.py --index

# 全市场个股前复权日 K 增量（读本地分红缓存，本季未刷新才打 fhsp）
python scripts/sync_sohu_data.py --all
```

**分批跑**（单次太久时）：

```bash
python scripts/sync_sohu_data.py --all --offset 0    --limit 1500
python scripts/sync_sohu_data.py --all --offset 1500 --limit 1500
python scripts/sync_sohu_data.py --all --offset 3000 --limit 9999
```

**常用参数**

| 参数 | 说明 |
|------|------|
| `--instruments 600600.SH` | 只更新指定股票 |
| `--stock-full --start 2020-01-01` | 单票全量重建 |
| `--force-fhsp` | 强制刷新分红缓存（分红季偶尔跑一次） |
| `--index-full --index-instruments 000300.SH` | 指数全量重拉修数 |

分红缓存已灌满后，日常 `--all` 基本只拉行情，速度快。

### 2.2 月表（日 K 更新后自动触发）

`sync_sohu_data.py` 在写入 `kline_qfq` 后会自动调用月表增量更新。若需手工全量重建：

```bash
python scripts/build_kline_qfq_monthly.py --rebuild
```

### 2.3 Moma 备选数据源

搜狐不可用时，可用 Moma 更新日 K：

```bash
python scripts/update_kline_qfq.py
python scripts/update_kline_qfq.py --instruments 600600.SH
```

基础元数据（非每日必须）：

```bash
python scripts/sync_moma_data.py --stock-list
python scripts/sync_moma_data.py --index-bar    # Moma 指数（已推荐改用搜狐 --index）
```

### 2.4 交易日历

新股/节假日变更后执行：

```bash
python scripts/sync_exchange_calendar.py
```

## 三、量化选股（quant_picker）

### 3.1 日常选股

```bash
# Top 50
python scripts/quant_picker.py --t_date 2026-06-12 --top_k 50

# 观盘池（四绿灯 + 大盘过滤，弱势日可能为空）
python scripts/quant_picker.py --t_date 2026-06-12 --watchlist --top_k 50
```

输出：

- `results/top_{N}_stocks_{日期}.json`
- `results/watchlist_{日期}.json`（`--watchlist` 模式）
- 日志：`logs/quant_picker_*.log`

### 3.2 内置优化（2026-06 验证后接入）

| 机制 | 行为 |
|------|------|
| **大盘择时 P0** | 环境分 &lt;30：Top-K 减半；30~60：仅 `final_score` 120~150；≥60：正常 |
| **分数修正 P1** | 反弹分过高、W 底、月线放量、总分过热等减分 |
| **Watchlist P2** | 四绿灯 + 上证 MA20 上或沪深300 近20日≥0；排除涨停锚点、W 底 |
| **月线量能 P3** | 底部放量 bonus 减半 |

选股前可先查看大盘环境分：

```bash
python scripts/market_env_score.py --date 2026-06-12
```

### 3.3 单票调试

```bash
python scripts/quant_picker.py --debug_stock 002082.SZ --t_date 2026-04-17
```

## 四、选股后分析工具

| 脚本 | 用途 | 示例 |
|------|------|------|
| `rerank_top_stocks.py` | 按锚点后 N 日涨幅/涨停重排 JSON 列表 | `python scripts/rerank_top_stocks.py results/top_50_stocks_2026-06-03.json` |

## 五、建议的日常流程（交易日）

```text
1. sync_sohu_data.py --index          # 指数
2. sync_sohu_data.py --all            # 全市场日 K + 月表
3. market_env_score.py                # 看今日环境分
4. quant_picker.py --t_date 今日 --top_k 50
5. quant_picker.py --t_date 今日 --watchlist --top_k 50   # 可选
```

周末或复盘时：运行 `validate_selections.py` + `report_selection_stats.py` 更新验证报告。

## 六、一次性维护

### 全市场分红缓存灌库

首次部署或分红季全量刷新（约 50~90 分钟）：

```bash
python scripts/_tmp_refresh_dividends.py
python scripts/_tmp_refresh_dividends.py --skip-existing   # 断点续跑
```

完成后可删除 `_tmp_refresh_dividends.py`、`_tmp_retry_dividends.py`。

### 历史选股重跑 + 验证

```bash
python scripts/quant_picker.py --t_date 2026-04-17 --top_k 50
# 对每个历史日重复后：
python scripts/validate_selections.py --out output/selection_validation_regen.csv
python scripts/report_selection_stats.py
```

## 八、常见问题

**Q：`sync_sohu_data.py` 报 `No module named 'loguru'`**  
A：未激活 `openclaw_env`，或在该环境中 `pip install loguru`。

**Q：指数/个股拉取失败**  
A：检查网络与 `q.stock.sohu.com` 可达性；失败股票可用 `--instruments` 单票重试。

**Q：`quant_picker` 候选很少或 watchlist 为空**  
A：查看日志中的「大盘环境分」；弱势/中性日本身会收紧输出，属预期行为。

**Q：`kline_qfq` 与 Moma 前复权不一致**  
A：用 `sync_sohu_data.py --instruments 代码 --stock-full` 全量重建；分红缓存需完整。
